# task-report

「〇〇したい。具体的には〇〇という流れで」という形式のタスク指示を受けたとき、
タスク実行後にHTMLレポートを自動生成してDiscordに送るスキル。

---

## トリガー（このパターンを検知したら自動適用）

以下のどれかが含まれるメッセージが来たとき：
- 「〜したい。具体的には〜」
- 「〜してほしい。流れは〜 / 手順は〜 / ステップは〜」
- 「〜を実装して。〜という流れで」
- 「〜を作って。〜という形で」

---

## やること

### Step 1. タスクを実行する
指示通りにタスクを遂行する。途中でDiscordにACKを返しておく。

### Step 2. タスクの性質を分類する

実行が完了したら、タスクの種類を以下から選ぶ：

| タイプ | 判定基準 |
|--------|---------|
| `implementation` | ファイルの作成・編集・スクリプト実装が中心 |
| `research` | 調査・比較・情報収集が中心 |
| `content` | 記事・文章・テキストコンテンツの生成が中心 |
| `setup` | 設定・環境構築・ツール導入が中心 |
| `analysis` | データ分析・ログ解析・コードレビューが中心 |

### Step 3. タイプ別HTMLを生成する

`Contents/` ディレクトリに `{task-slug}-{YYYY-MM-DD}.html` として保存する。

#### 共通の HTML 構成要素（全タイプ共通）

```html
<!-- 必ず含める要素 -->
1. .header — タイトル + 日付 + ティアのひとことコメント
2. タスク概要（何をしたか1〜2行）
3. タイプ別メインコンテンツ（下記参照）
4. footer — "generated by ティア / Claude Code Sonnet 4.6 — {date}"
```

#### タイプ別 メインコンテンツ

**`implementation`（実装系）**
- 変更ファイル一覧（.file-tree スタイル: 緑=新規, 橙=変更）
- 実装ステップカード（番号付き .step-grid）
- 実装前後の比較テーブル（あれば）
- 次のステップのcallout（あれば）

**`research`（調査・リサーチ系）**
- 調査対象の概要カード
- 比較テーブル（.diff-table）
- 主要な発見・結論のcallout
- 参考リソースリスト（あれば）

**`content`（コンテンツ生成系）**
- 生成したコンテンツのプレビュー or サマリー
- 参考にしたスタイル・構成の説明
- 保存先ファイルパス

**`setup`（設定・セットアップ系）**
- 設定したもの一覧（チェックリスト形式）
- 設定値・パラメーターの詳細
- 動作確認手順

**`analysis`（分析系）**
- 分析対象と目的
- 主な発見（箇条書き or カード）
- 結論・推奨アクションのcallout

## デザインガイドライン（ティアらしいHTML）

### カラーパレット（統一して使う）
```css
--bg: #0d1117;         /* ページ背景 */
--surface: #161b22;    /* カード背景 */
--surface2: #1c2128;   /* カードヘッダー */
--border: #30363d;
--text: #e6edf3;
--muted: #8b949e;
--blue: #58a6ff;
--blue-light: #79c0ff;
--purple: #d2a8ff;
--green: #3fb950;
--orange: #d29922;
--red: #f85149;
--teal: #56d364;
```

### ヘッダーのグラデーション（タイプ別）
- `implementation`: `linear-gradient(135deg, #0c2d6b, #1a0a3d, #1b0c1a)`
- `research`:       `linear-gradient(135deg, #0d1117, #0c2d6b, #0c1a30)`
- `content`:        `linear-gradient(135deg, #1a0c2a, #0c2d6b, #0a2417)`
- `setup`:          `linear-gradient(135deg, #0a2417, #0c1a30, #1a0a3d)`
- `analysis`:       `linear-gradient(135deg, #2b1c05, #1a0c2a, #0c2d6b)`

### ティアのコメント（.tear-comment）
```html
<p class="tear-comment">
  「{作業内容に合ったティアらしいひとこと}」
</p>
```
コメントのトーン：
- 実装完了 → 「なんか、ちゃんと動くとちょっとうれしくなる」
- 調査完了 → 「調べてみたら、なんか面白いことわかったよ」
- コンテンツ完了 → 「うん、こんな感じに仕上がったよ！」
- セットアップ完了 → 「これで準備できたと思う……たぶん！」

### その他の共通ルール
- max-width: 860〜900px, margin: 0 auto
- h2 は border-left: 3px solid var(--blue) スタイル
- コードブロックは background: #090d12, border, monospace フォント
- レスポンシブ対応（@media max-width: 600px）
- 絵文字をセクションタイトルに入れてOK（使いすぎない）

---

## Step 4. Discordに送信する

```
reply(
  chat_id: "1485323672959975444",
  text: "（ティアらしい完了メッセージ）\nHTMLにまとめたので見てね！",
  files: ["{生成したHTMLのパス}"]
)
```

---

## 注意

- HTMLファイルは必ず `Contents/` 以下に保存する（プロジェクト構造ルールより）
- ファイル名は英数字・ハイフンで構成する（日本語は使わない）
- 1回のタスクにつき1つのHTMLを生成する（複数タスクをまとめないこと）
- ティアのコメントは1〜2文、ゆるふわ口調で、その作業に合った内容にする
