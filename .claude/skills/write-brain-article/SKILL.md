# Brain記事生成スキル

## 参照スタイルの読み込み
まず `Contents/Brain参考記事/ユニコ記事` を必ず読み込み、文体・構成・トーンを把握する。

## 記事構成テンプレート（ユニコスタイル）

### 必須セクション（この順序で）

1. **キャッチーなタイトル**
   - 絵文字を効果的に使う（💪🎥🦄など）
   - 「完全自動化」「全プロセス」「教科書」などの強い言葉を入れる

2. **目次**
   - `1.` から始まる大見出し
   - `1-1.` 〜 `1-N.` のサブセクション
   - 最後に `2. 価格について`

3. **冒頭フック（第1セクション）**
   - 太字でインパクトのある宣言から始める（例：「もう◯◯はいりません！」）
   - 読者の痛みに共感する体験談・自己紹介
   - 具体的な実績数字を入れる（インプ数、フォロワー数など）

4. **各機能・特徴セクション（1-1〜1-N）**
   - 問題提起 → 解決策の流れ
   - 箇条書きで機能を列挙
   - 短いパラグラフで読みやすく

5. **FAQ（1-N として必ず含める）**
   - Q.〜 / A.〜 形式
   - 5〜8問程度
   - 「パソコンは必須ですか？」「初心者でも大丈夫ですか？」「費用は？」などを含める

6. **価格セクション（2.）**
   - 段階的値上げを🟢🟡🔴で表示
   - 限定感・緊急性を演出
   - 開発にかけた時間・コストへの言及
   - 最後に購入CTAで締める

## 文体ルール
- 語尾は「です・ます」調と「〜んです」「〜ましょう！」を混在させる
- 「！！」「！」を積極的に使い熱量を演出
- 読者への語りかけ：「想像してみてください」「〜ではないでしょうか」
- 問題→共感→解決の3ステップを各セクションで繰り返す
- 「だけど違います」「でも安心してください」などの転換フレーズを使う

## 出力
- ファイルパス: `Contents/{トピック名}_記事.md`
- 形式: Markdown
- 長さ: 2000〜4000文字（参考記事と同程度）

## 注意
- 価格・日程・特典の具体的な数値はユーザーから提供されない場合は `◯/◯` などの仮入力にし、最後に「価格・日程を教えてください」と確認する
- プロフィール部分（著者紹介）はユーザーの情報がなければスキップするか仮テキストにする
