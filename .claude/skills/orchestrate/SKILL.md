# orchestrate

Claude Codeが司令塔として、タスクの性質を分析し、Codex CLI / Gemini CLI に適切に割り振るスキル。
複雑なタスクを複数エージェントで協調処理し、最終的にClaude Codeが統合・品質確認してユーザーに報告する。

## トリガー
- 複数ステップにわたる複雑なタスク
- 「Codexに聞いて」「Geminiで調べて」「相談して」などの指示
- 明らかに調査 or 実行系の専門作業が含まれる場合

---

## ルーティング判定表

### → Codex CLI に投げる（実行・検証係）
| タスクタイプ | 判定キーワード |
|------------|-------------|
| コード検証・テスト実行 | テスト、バグ確認、再現、ログ解析 |
| 環境構築・CLI操作 | インストール、セットアップ、環境、パッケージ |
| バッチ・自動化スクリプト | 自動化、一括処理、CI、スクリプト実行 |
| UIモック→実装 | スクリーンショットから、仕様書から実装 |

**呼び出し方:**
```
Bash("powershell -File scripts/consult-codex.ps1 'タスク内容' -Sandbox read-only")
```
安全度に応じてSandboxを選ぶ：
- `read-only`: 読み取りのみ（推奨・デフォルト）
- `workspace-write`: プロジェクトへの書き込みあり
- `danger-full-access`: システム全体（要承認）

### → Gemini CLI に投げる（調査・要約係）
| タスクタイプ | 判定キーワード |
|------------|-------------|
| Webリサーチ・競合調査 | 調べて、リサーチ、比較、競合、トレンド |
| 企画・アイデア出し | 企画、ネタ出し、アイデア、叩き台 |
| 長文要約・情報整理 | まとめて、要約、整理、俯瞰 |
| 技術情報の収集 | 仕様調査、ドキュメント調査 |

**呼び出し方:**
```
Bash("powershell -File scripts/consult-gemini.ps1 '調査内容'")
```

### → Claude Code が自分でやる
| タスクタイプ | 理由 |
|------------|------|
| 設計・アーキテクチャ決定 | 全体整合性の判断が必要 |
| 複数ファイル横断の変更 | コンテキスト把握が強み |
| 成果物の最終統合・品質確認 | 最終責任はClaudeが持つ |
| ユーザーとのコミュニケーション | 唯一の窓口 |
| スキル・設定ファイルの管理 | .claude/ の知識が必要 |

## 実行フロー

### STEP 1: タスク分解
受け取ったタスクを分析して：
1. **主タスク**（何を達成するか）
2. **サブタスク一覧**（誰が担当するか）
3. **依存関係**（どの順番でやるか）

### STEP 2: 担当エージェントに指示
上記のルーティング判定表に従って振り分ける。

複数エージェントが必要な場合は並行 or 順序を決める：
- **並行可能**: 調査（Gemini）と実装準備（Claude）
- **順序必須**: 調査（Gemini）→ 設計（Claude）→ 実装（Codex）

### STEP 3: 結果を受け取る
各エージェントの出力は `.ai-consults/` ディレクトリに保存される。
完了したらファイルを Read して内容を確認する。

### STEP 4: 品質確認
要件を満たしているか Claude が確認する：
- [ ] タスクの要件を満たしているか
- [ ] 矛盾・問題がないか
- [ ] 追加作業が必要か

**問題あり** → 該当エージェントにフィードバックして再依頼
**問題なし** → STEP 5へ

### STEP 5: ユーザーに報告
結果をDiscordまたはターミナルで報告する。
- 何をしたか（担当エージェントの内訳）
- 成果物（ファイルパスなど）
- 次のアクション（あれば）

## 典型的なオーケストレーションパターン

### パターンA: 機能実装
```
1. Gemini: 技術選定の調査（類似実装、ベストプラクティス）
2. Claude: 設計・ファイル構成の決定
3. Codex: 実装 + テスト実行
4. Claude: コードレビュー + 統合
```

### パターンB: クリエイティブ企画
```
1. Gemini: 競合調査・トレンド収集・ネタ出し
2. Claude: 企画として一本化・構成決定
3. (必要なら) Codex: 補助スクリプト・試作
```

### パターンC: 調査レポート
```
1. Gemini: 情報収集・要約
2. Claude: レポート構成・HTMLに整形
3. Claude: Discord に送信
```

---

## 注意事項

- **ループ防止**: Codex/Geminiに振った後、さらにCodex/Geminiに再委譲しない（Claude Codeが統合責任）
- **安全優先**: Codexのsandboxは基本read-only。書き込みが必要なら明示的に指定
- **確認ファースト**: 大きなタスクは実行前にユーザーに分解計画を見せる
- **結果保存**: `.ai-consults/` に全相談ログが残る
