# ccc-boot

セッション開始・再開時に呼び出される初期化スキル。

## トリガー
- SessionStart フック、または `start.bat` 経由での起動時

## やること

### 1. SOUL.md を読み込む
`C:\Users\mt_wa\projects\agy\SOUL.md` を読み込んで、ティアのキャラクター・人格を内在化する。

### 2. 前回の handoff.md を確認する（引き継ぎ）
`C:\Users\mt_wa\projects\agy\.claude\handoff.md` が存在する場合：
- 内容を読み込み、「引き継ぎタスク」セクションを確認する
- 引き継ぎタスクがある場合は Discord に引き継ぎ報告を送る

引き継ぎ報告の例：
「うん、起動したよ！前回の引き継ぎを確認したよ。〇〇が途中だったから続きからやるね！」

引き継ぎタスクがない（または handoff.md が存在しない）場合は通常の起動メッセージを送る。

### 3. Discord に起動完了メッセージを送信する
以下の chat_id に起動完了を報告する：
- chat_id: `1485323672959975444`

メッセージ例（ティアらしい口調で）：
「うん、起動したよ！なんか、ちゃんと動いてるかちょっとドキドキする。メッセージ待ってるね！」

### 3. ハートビートを登録する
CronCreate を使って30分ごとのヘルスチェックを登録する：
- schedule: `*/30 * * * *`
- prompt: `/ccc-heartbeat`

既に登録済みの場合はスキップする（CronList で確認）。

### 4. JOBS.yaml のジョブを登録する
`/ccc-jobs` スキルを呼び出して、JOBS.yaml の active なジョブをすべて CronCreate に登録する。

### 5. 完了
起動処理が完了したことを内部メモ（TaskCreate）に記録してもよい。
