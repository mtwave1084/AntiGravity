# Brain記事執筆ルール

## スタイルの基本原則
- 参照記事: `Contents/Brain参考記事/ユニコ記事`
- プラットフォーム: Brain（https://brain-market.com）向けセールス記事
- ターゲット: AIエージェント・動画制作・Claude Code に関心のある日本語話者

## 構成パターン
```
タイトル（絵文字入り・インパクト重視）
↓
目次（番号付き、1-1〜1-N のサブセクション構造）
↓
1. メインセクション（問題提起＋解決策＋実績）
  1-1. 読者の悩みを言語化
  1-2. 著者の自己紹介・実績
  1-3〜1-N. 機能・特徴・特典の詳細
  1-N. FAQ
↓
2. 価格（🟢🟡🔴 段階値上げ＋購入CTA）
```

## トーン・文体
- **熱量**: 「！！」多用、太字での強調を積極的に使う
- **共感**: 読者の痛みを「〜ではないでしょうか」で言語化してから解決策を提示
- **転換**: 「だけど違います」「でも安心してください」「もう必要ありません」
- **具体性**: フォロワー数・インプ数・受講生数などの実績数字を入れる
- **CTA**: 各セクションの末尾で軽く行動を促し、最後の価格セクションで強くCTAを入れる

## 禁止事項
- 過度に技術的・説明的な文体（セールス感を損なう）
- 箇条書きだけで構成された無機質なセクション（体験談・語りかけを混ぜる）
- 価格・日程の無断確定（仮値で書き、ユーザーに確認する）

## ファイル命名規則
`Contents/{記事トピック}_記事.md`
例: `Contents/AntiGravity_Remotion_記事.md`
