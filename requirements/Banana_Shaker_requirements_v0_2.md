# Banana Shaker 要件定義書 v0.2

## 1. システム概要

- **名称**：Banana Shaker
- **目的**：
  Nanobanana / Nanobanana Pro（以下まとめて Nanobanana 系）を、
  - プリセット管理
  - 再利用しやすい履歴・ギャラリー
  - 入力画像を使った生成
  で「自分がストレスなく使うため」の **個人用画像生成支援Webアプリ** として構築する。

- **利用形態**：
  - PCブラウザ前提
  - ログイン必須
  - v1は管理者（=本人）専用
  - 将来、一般公開し「UI提供＋利用者自身のAPIキー」での利用も視野

---

## 2. 想定ユーザー・ユースケース

### 2.1 想定ユーザー

- **主ユーザー**：開発者本人（Nanobananaヘビーユーザー）
- **将来的ユーザー**（v2以降）：
  - 画像生成を日常的に行うクリエイター
  - キャラクターデザインを量産したい人
  - マンガ・同人系ラフをAIで出したい人

### 2.2 主要ユースケース（v1視点）

1. **プリセット管理**
   - よく使うプロンプト＋パラメータをプリセットとして保存
   - 一覧から選択し1クリックで生成

2. **通常生成**
   - プロンプト＋パラメータをその場で入力し生成
   - プリセットを読み込んで微調整して生成

3. **入力画像を利用した生成**
   - 画像をアップロードし、構図や衣装・背景などを差し替えて生成（※v1ではUI未実装、バックエンドのみ対応）

4. **履歴・ギャラリー**
   - 過去の生成結果を一覧で閲覧
   - プロンプト・パラメータを確認し、同じ条件で再生成（Remix）

## 3. 機能要件（v1 実装済み）

### 3.1 認証・ユーザー管理

- [DONE] ログイン機能（メール＋パスワード）
  - `next-auth` (Auth.js) Credentials Provider を使用
- [DONE] 認証状態の維持（セッション）
- [DONE] 未ログイン時はログイン画面へリダイレクト
- [FUTURE] 外部認証（Google / X / GitHub 等）

### 3.2 APIキー管理

- [DONE] Nanobanana / Nanobanana Pro の APIキー入力フォーム
- [DONE] APIキーの保存・更新
  - データベースに暗号化して保存（AES-256-CBC）
- [DONE] 生成実行時に該当サービスのキーを使用
- [DONE] 画面上は伏字表示＋「表示/隠す」切替

### 3.3 画像生成機能

#### 3.3.1 通常生成

- 入力項目：
  - 使用サービス：`Nanobanana / Nanobanana Pro`
  - 生成枚数：1〜4枚
  - アスペクト比：1:1, 2:3, 3:2, 3:4, 4:3, 16:9
  - 解像度：`1k / 2k / 4k`
  - シード：数値 or auto（null）
  - メインプロンプト（必須）
  - ネガティブプロンプト（任意）
- 動作：
  - Server Action `generateImage` を呼び出し
  - Google GenAI SDK (`gemini-2.5-flash-image` / `gemini-3-pro-image-preview`) を使用
  - 完了後にギャラリーへリダイレクト（v1仕様）

#### 3.3.2 入力画像付き生成

- [PARTIAL] バックエンド（`nanobanana.ts`）は対応済みだが、UI実装はv1.1以降へ持ち越し

### 3.4 プリセット機能

- プリセット項目：
  - タイトル、タグ
  - プロバイダ、タスク種別
  - アスペクト比、解像度、生成枚数、シード
  - プロンプト、ネガティブプロンプト
- 機能：
  - [DONE] 一覧表示（カード形式）
  - [DONE] 新規作成・編集・削除（モーダルUI）
  - [DONE] 「このプリセットで生成」ボタン

### 3.5 ギャラリー・履歴

- 記録する内容：
  - 生成日時、使用サービス、プロンプト等全パラメータ
  - 生成された画像（Base64）
- 機能：
  - [DONE] サムネイル一覧表示
  - [DONE] 詳細モーダル表示
  - [DONE] モーダルから「Remix」（生成画面へ遷移してパラメータセット）

## 4. 画面仕様（v1 実装済み）

### 4.1 共通レイアウト

- 左サイドバー（`AppSidebar`）：ナビゲーション（Dashboard, Presets, Generate, Gallery, Settings）
- 上部ヘッダー（`SiteHeader`）：ページタイトル表示
- メインコンテンツエリア

### 4.2 ログイン画面

- メールアドレス・パスワード入力フォーム
- エラー表示対応

### 4.3 ダッシュボード

- 最近のプリセット（最大5件）
- 最近の生成画像（最大8件）

### 4.4 プリセット画面

- プリセット一覧（グリッド表示）
- 新規作成ボタン（モーダル起動）
- 各カードに編集・削除・使用ボタン

### 4.5 生成画面

- プリセット読み込みプルダウン
- 各種パラメータ入力フォーム
- 生成ボタン（ローディング表示あり）

### 4.6 ギャラリー画面

- 全生成画像のグリッド表示
- クリックで詳細モーダル表示

### 4.7 設定画面

- APIキー管理フォーム（Nanobanana / Pro）

## 5. データモデル（SQLite）

Prisma から `better-sqlite3` + 生SQLマイグレーションに変更。

### テーブル定義

- **User**: ユーザー情報（ID, Email, Password, Name）
- **ApiKey**: APIキー（UserId, Provider, EncryptedKey）
- **Preset**: プリセット設定
- **GenerationJob**: 生成ジョブ履歴
- **Image**: 生成画像データ（Base64保存）

---

## 6. 技術スタック（確定）

- **フロント / API**：Next.js 15 (App Router) + TypeScript
- **UI**：Tailwind CSS + shadcn/ui
- **認証**：Auth.js (v5)
- **DB**：SQLite (`better-sqlite3`)
  - ※ Prisma は環境依存の問題により不採用
- **画像生成**：Google GenAI SDK (`@google/generative-ai`)
- **デプロイ**：ローカル実行 (`npm run dev`) 推奨
  - ※ ビルド時のネイティブモジュール問題があるため、v1では dev モードでの利用を前提とする

---

## 7. 今後の課題（v1.1 / v2）

- **ビルド問題の解決**: `better-sqlite3` のバンドル設定を調整し、`npm run build` を通す
- **入力画像対応**: UIに入力画像アップロード機能を追加
- **画像ストレージ**: Base64保存からファイルシステムまたはクラウドストレージへの移行
- **非同期生成**: 生成中の進捗表示とバックグラウンド処理化
