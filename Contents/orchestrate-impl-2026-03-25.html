⚙ ラッパースクリプトの呼び出し方

Codexに相談する（Claude Code 内から）
# read-only sandbox（デフォルト・推奨）
Bash("powershell -File scripts/consult-codex.ps1 'このコードのテストを実行して結果を教えて'")

# 書き込みあり
Bash("powershell -File scripts/consult-codex.ps1 '依存パッケージを追加して' -Sandbox workspace-write")
Geminiに相談する（Claude Code 内から）
Bash("powershell -File scripts/consult-gemini.ps1 'このジャンルの競合を5つ調べてトレンドをまとめて'")

結果は .ai-consults/codex-{timestamp}.md / gemini-{timestamp}.md に保存される。Claude が Read して内容を確認・統合する。

consult-gemini.ps1 経由 → .ai-consults/ に結果保存

4
Claude: 品質確認 + フィードバック
要件を満たしているか確認。問題があれば再依頼。
↓ 問題なし
5
Claude: 統合 + ユーザーに報告
Discord に完了報告・成果物・次のアクションを送信
⚠ 注意：Codex/Gemini のインストール確認

ラッパースクリプトは codex / gemini コマンドがPATHにある前提。

未インストールの場合: npm i -g @openai/codex / npm i -g @google/generative-ai

Geminiのフラグ（-p）は実際のバージョンで gemini --help で要確認。

✓ 段階的な導入推奨

最初は明示コマンドで手動呼び出しから。うまく回ってから CLAUDE.md に自動判断ルールを追加していくのが安全。完全自動化は無限ループのリスクがある。
