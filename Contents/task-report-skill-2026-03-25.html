🎯 このスキルが何をするか

🔍
トリガーパターンの検知

以下のパターンを含むメッセージを受信したとき、タスク完了後に自動でHTMLレポートを生成する：

✓「〇〇したい。具体的には〇〇という流れで」
✓「〇〇してほしい。流れは / 手順は / ステップは〇〇」
✓「〇〇を実装して。〇〇という流れで」
✓「〇〇を作って。〇〇という形で」

🏷 タスクタイプと生成されるHTML

implementation
ファイル作成・編集・スクリプト実装が中心
  • 変更ファイルツリー（緑=新規/橙=変更）
  • 実装ステップカード
  • 実装前後の比較テーブル
research
調査・比較・情報収集が中心
  • 調査対象の概要カード
  • 比較テーブル
  • 主要な発見・結論のcallout
content
記事・文章・テキスト生成が中心
  • 生成コンテンツのプレビュー
  • 参考スタイルの説明
  • 保存先ファイルパス
setup
設定・環境構築・ツール導入が中心
  • 設定項目チェックリスト
  • 設定値・パラメーター詳細
  • 動作確認手順
analysis
データ分析・ログ解析・コードレビューが中心
  • 分析対象と目的
  • 主な発見（箇条書き or カード）
  • 結論・推奨アクションのcallout

🔁 実行フロー

1
Discord メッセージ着信
「〜したい。具体的には〜」パターンを検知
↓ 即時ACK
2
タスクを実行する
指示通りに作業を完了させる
↓
3
タスクタイプを判定
implementation / research / content / setup / analysis
↓
4
タイプ別HTMLを生成
Contents/{task-slug}-{YYYY-MM-DD}.html に保存
↓
5
Discord に送信
ティアらしい完了メッセージ + HTML添付
