1. CCCBot とは何か

CCCBot は Claude Code Channels を土台にした自律エージェントフレームワーク。Discord/Telegram ボットとして動作するのではなく、Claude Code セッション自体を自律的に稼働させ続ける ための設定・スキル・シェルスクリプト群。

💡 核心の考え方
Claude Code Channels の MCP プラグインはブリッジに過ぎない。CCCBot は Claude Code セッションを「常駐サービス」として動かし、セッション再起動・ハートビート監視・スケジュール実行をシェルスクリプトで自動化する。

🏗 CCCBot のアーキテクチャ

  • 言語: Bash / PowerShell（Node.js 不要）
  • 依存: git, claude CLI, curl のみ
  • Discord/Telegram 同時対応（MCP プラグイン経由）
  • PID ロックによるセッション重複防止
  • MCP 切断 → 自動再起動
  • SOUL.md でキャラクター・人格定義
  • JOBS.yaml でスケジュールタスク管理

🔌 あなたの現在の実装

  • 言語: TypeScript (Bun)
  • 依存: @modelcontextprotocol/sdk, discord.js
  • MCP サーバーとして Claude Code にメッセージを橋渡し
  • Permission Relay 実装済み（claude/channel/permission）
  • ペアリング・アクセス制御完備
  • 自律稼働の仕組みはまだない

3. CCCBot のメッセージ処理フロー

1
Discord メッセージ着信
MCP プラグイン (server.ts) がゲート判定 → Claude Code に通知
↓
2
ccc-channel-task スキル起動
Claude が自動的にスキルを呼び出す（CLAUDE.md で定義）
↓ 即時（最優先）

↓

短タスク → 同期実行 ／ 長タスク → run_in_background=true でサブエージェント
↓
5

push 通知のため新規メッセージとして送信

4. セッション永続化の仕組み

start.sh / start.bat の役割

start.sh（概要）
WORKSPACE="$HOME/.cccbot"
PID_FILE="$WORKSPACE/.claude/ccc-session.pid"

# PID ロック — 二重起動防止
if [ -f "$PID_FILE" ] && kill -0 $(cat "$PID_FILE") 2>/dev/null; then
  echo "セッションはすでに起動中"
  exit 1
fi

# セッション再開を試みる（前回の続き）
claude "/ccc-boot" --continue \
  --channels discord,telegram \
  --remote-control

重要なのは --continue フラグ。前回のセッションコンテキストを引き継いで起動する。

SessionStart フック

.claude/settings.json
{
  "permissions": {
    "defaultMode": "bypassPermissions",
    "allow": ["WebSearch", "Read(.claude/**)", "Edit(.claude/**)", "Write(.claude/**)"],
    "deny": [
      "Bash(rm -rf /*)", "Bash(git push --force*)",
      "Bash(git reset --hard*)", "Bash(npm publish*)"
    ]
  },
  "hooks": {
    "SessionStart": [{
      "matcher": "startup|resume",
      "hooks": [{
        "type": "command",
        "command": "bash .claude/scripts/session-start-hook.sh"
      }]
    }]
  }
}

セッション開始・再開のたびに ccc-boot スキルが自動起動される。

5. ハートビートと自動再起動

ccc-heartbeat の動作

  • 30分ごとに CronCreate で登録
  • run_in_background=true でサブエージェント起動
  • MCP に軽量テスト送信（reply ツール）
  • 疎通失敗 → restart-session.sh 起動
  • 問題なければ無音（通知しない）

restart-session.bat（Windows）

概要
timeout /t 3 /nobreak

set /p PID=<"%USERPROFILE%\.cccbot\.claude\ccc-session.pid"

# 安全確認して kill
taskkill /F /PID %PID%

# 新しいセッションを detached で起動
powershell -WindowStyle Hidden -Command ^
  "Start-Process 'start.bat' -WindowStyle Hidden"

CLAUDE.md に自律動作の指示を追加

Discordからメッセージが来たとき ccc-channel-task スキルを自動起動するよう指示する。

## Discord メッセージ処理
Discordからメッセージが届いたら必ず以下の順で対応する：
1. 即座に確認メッセージを返信する（"了解、調べます" など）
2. 短いタスクは同期実行
3. 長いタスクは run_in_background=true のサブエージェントで実行
4. 完了したら新規メッセージで結果を報告する

launcher スクリプトを作成（Windows: start.bat）

@echo off
set WORKSPACE=%USERPROFILE%\.cccbot
set PID_FILE=%WORKSPACE%\.claude\ccc-session.pid

:: PID ロック確認
if exist "%PID_FILE%" (
  set /p OLD_PID=<"%PID_FILE%"
  tasklist /FI "PID eq %OLD_PID%" 2>nul | find "%OLD_PID%" >nul
  if not errorlevel 1 (
    echo セッション稼働中: PID %OLD_PID%
    exit /b 0
  )
)

cd /d "%WORKSPACE%"
:: セッション起動（--continue で前回から継続）
claude "/ccc-boot" --continue --channels discord

8. Permission Relay の動作（実装済み詳細）

あなたの server.ts はすでに完全実装済み。動作フローは以下の通り：

1
Claude Code がツール実行前に許可が必要と判断
↓
2
MCP に notifications/claude/channel/permission_request を送信
request_id, tool_name, description, input_preview を含む
↓
3
server.ts が allowFrom の全ユーザーの DM に転送
"🔐 Permission request [xxxxx]
yes xxxxx で許可 / no xxxxx で拒否"
↓
4
ユーザーが "yes abcde" または "no abcde" と返信
↓
5
PERMISSION_REPLY_RE でパース → notifications/claude/channel/permission を emit
Claude Code が allow/deny を受け取って処理再開
⚠ セキュリティ注意点
server.ts は allowFrom（ペアリング済みユーザー）のみからの permission reply を受け付ける。ゲートを通過した送信者のみ信頼するため、なりすましによる悪用は防がれている。

10. 注意点・制限事項

📌 Claude Code Channels はリサーチプレビュー
v2.1.81 時点でリサーチプレビュー段階。仕様変更の可能性あり。CCCBot も「Anthropic 非公認・コミュニティプロジェクト」と明示している。
🔒 permissions の defaultMode に注意
bypassPermissions にすると全ツールが無確認で実行される。Permission Relay を活用するなら allowEdits にして危険な操作はDiscordから確認するフローが安全。
